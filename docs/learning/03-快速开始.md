# 03. 快速开始

让我们快速运行项目，看看效果！

## 🚀 第一步：获取项目代码

### 方式 1: 从 GitHub 克隆（推荐）

```powershell
# 克隆项目
git clone https://github.com/JiashuaiXu/winUI-image-resolver.git

# 进入项目目录
cd winUI-image-resolver
```

### 方式 2: 下载 ZIP

1. 访问 [GitHub 仓库](https://github.com/JiashuaiXu/winUI-image-resolver)
2. 点击 "Code" → "Download ZIP"
3. 解压到本地目录

## 🔧 第二步：还原依赖

项目使用 NuGet 包管理依赖，需要先还原：

```powershell
# 在项目根目录执行
dotnet restore
```

**说明**: 这会下载所有需要的 NuGet 包（ExifLib、CsvHelper、WindowsAppSDK 等）

## 🏗️ 第三步：编译项目

```powershell
# 编译项目
dotnet build
```

**预期输出**:
- 显示一些警告（正常，不影响运行）
- 最后显示 "Build succeeded"
- 生成的文件在 `bin/Debug/` 目录

## ▶️ 第四步：运行应用

### 方式 1: 使用 dotnet run

```powershell
dotnet run
```

### 方式 2: 直接运行可执行文件

```powershell
# 进入输出目录
cd bin\Debug\net8.0-windows10.0.19041.0\win-x64

# 运行程序
.\ImageResolver.exe
```

### 方式 3: 使用 Visual Studio

1. 打开 `ImageResolver.csproj`
2. 按 `F5` 运行（或点击"启动"按钮）

## 🎯 第五步：测试应用

1. **启动应用**
   - 应该看到一个现代化的窗口界面

2. **选择文件夹**
   - 点击 "Select Folder" 按钮
   - 选择包含图片的文件夹（项目中的 `pic/` 文件夹有测试图片）

3. **查看结果**
   - 左侧显示图片列表
   - 点击图片查看详细信息
   - 右侧显示 EXIF 数据

4. **导出 CSV**
   - 点击 "Export to CSV" 按钮
   - 选择保存位置
   - 用 Excel 打开查看

## 🐛 遇到问题？

### 问题 1: 提示需要 Windows App Runtime

**解决方案**: 
- 项目已配置自包含部署
- 重新编译：`dotnet build`
- 运行：`dotnet run`

### 问题 2: 找不到项目文件

**解决方案**:
- 确保在项目根目录（包含 `ImageResolver.csproj` 的目录）
- 使用 `Get-Location` 检查当前目录
- 参考 [BUG-008](../bugs/008-dotnet-command-wrong-directory.md)

### 问题 3: 编译错误

**解决方案**:
1. 检查 .NET SDK 版本：`dotnet --version`（应该是 8.0.x）
2. 清理并重新编译：
   ```powershell
   dotnet clean
   dotnet restore
   dotnet build
   ```

### 问题 4: 应用启动后没有反应

**解决方案**:
- 检查是否有错误提示
- 查看 Windows 事件查看器
- 使用 `dotnet run` 查看控制台输出

```
├── MainWindow.xaml.cs    # 主窗口逻辑
├── Models/               # 数据模型
│   └── ImageInfo.cs
├── Services/              # 业务逻辑服务
│   ├── ImageInfoService.cs
│   └── CsvExportService.cs
├── Assets/               # 资源文件（图标等）
└── pic/                  # 测试图片
```
